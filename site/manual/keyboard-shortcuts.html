Keyboard Shortcuts

Default key bindings, keyboard profiles, and customization.

Keyboard Profiles

DMEdit comes with six keyboard profiles that map shortcuts to match editors you may already be familiar with:

  • Default — DMEdit's own layout, documented in the tables below.
  • VS Code
  • Visual Studio
  • JetBrains (IntelliJ, Rider, etc.)
  • Eclipse
  • Emacs

Switch profiles in the Keyboard section of Settings. The active profile changes all default shortcuts to match that editor's conventions.

Chord Shortcuts

Some shortcuts use a two-key chord: press the first key combination, release it, then press the second key combination within the chord timeout (default: 3 seconds). For example:

  • Ctrl+E, Ctrl+W — toggle word wrap
  • Ctrl+R, Ctrl+W — toggle whitespace

When you press the first key of a chord, DMEdit waits for the second key. If you don't press it in time, the chord is cancelled.

Customizing Shortcuts

You can override individual shortcuts in the Keyboard section of Settings. Your overrides are stored separately from the profile, so switching profiles doesn't lose your customizations. Set a binding to empty to unbind a command.

Each command can have up to two shortcut bindings (a primary and a secondary).
