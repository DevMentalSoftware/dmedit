Tabs & Windows

Working with multiple documents.

Opening Tabs

Each file you open gets its own tab in the tab bar at the top of the editor. You can have many files open simultaneously. Each tab independently maintains its own:

  • Scroll position
  • Caret position and selection
  • Undo/redo history
  • Find bar search term

New documents created with Ctrl+N also open in their own tab.

Close current tabCtrl+F4

If a tab has unsaved changes, you'll be prompted to save before closing.

Tab Indicators

Tabs show visual indicators for the document's state:

  • Unsaved changes dot — a bullet appears in the tab header when the document has been modified since the last save. It disappears when you save, or when you undo back to the saved state.
  • Read-only badge — shown when the file is in read-only mode (either because you toggled it or because the file is read-only on disk).
  • Conflict icon — appears when the file has been modified externally while you have unsaved changes. See the section below.

External File Changes

If another program modifies a file that you have open:

  • No unsaved edits — if Auto-Reload is enabled in Settings, the file is reloaded automatically. Otherwise, a conflict icon appears on the tab.
  • Unsaved edits — a conflict icon appears on the tab. You decide whether to reload the file (losing your edits) or keep your version.

See File Operations for details on Auto-Reload and Revert.

Window State

DMEdit remembers your window size, position, and whether it was maximized. These are restored the next time you launch the application.
