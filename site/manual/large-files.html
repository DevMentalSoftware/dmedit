Large Files

Working with multi-gigabyte files.

Opening Large Files

DMEdit can open files of virtually any size — including multi-gigabyte logs, data dumps, and other large text files. Files are loaded using a streaming approach that keeps memory usage low regardless of file size. You don't need to do anything special; just open the file as normal.

Scrolling

DMEdit uses a windowed rendering approach: only the portion of the document visible on screen (plus a small buffer) is held in memory and rendered at any time. This means scrolling through a 10 GB file feels the same as scrolling through a 10 KB file.

The dual-zone scroll bar is especially useful for large files:

  • Inner thumb — drag to jump proportionally through the document. Useful for getting to a rough position quickly.
  • Outer thumb — drag for a fixed-speed scan. Useful for reading through content at a consistent pace. The scan speed is adjustable in Settings.

Character-Wrap Mode

Some large files contain extremely long lines (for example, minified JavaScript or single-line JSON). When DMEdit detects a file that is both large (default: over 50 KB) and contains a line longer than 2,000 characters, it automatically switches to character-wrap mode.

In character-wrap mode:

  • Lines are wrapped at a fixed character width rather than at word boundaries.
  • Scrolling uses optimized math that stays fast regardless of line length.
  • The editor remains responsive even with lines that are millions of characters long.

The file size threshold for character-wrap mode is configurable in Settings under Char Wrap File Size (KB).

Performance

A few things to know when working with very large files:

  • Search works across the entire file. Match counting runs asynchronously so it doesn't block the editor.
  • Replace All processes the file in chunks with a progress indicator for very large documents.
  • Selecting large amounts of text is handled efficiently without copying the entire selection into memory until needed (for example, when you copy it to the clipboard).
  • Find and Replace operations near the end of a very large file work correctly and scroll to show the result.
